# Stock Analyst — 국내·해외 주식 종합 분석

티커 또는 회사명을 입력받아 **회사 개요 / 시세·재무 / 애널리스트 컨센서스 / 차트(기술적 분석) / 동종업계 비교 / 최근 뉴스 / 종합 의견** 7개 섹션으로 분석 리포트를 생성한다.

## 워크플로우

### 1. 종목 식별
사용자 입력에서 다음을 추출:
- **시장 판별**: 한글 회사명/6자리 숫자 → 국장 / 영문 1~5자 티커 → 미장 (모호하면 사용자에게 확인)
- **티커 정규화**:
  - 국장: 종목코드 6자리 (예: 삼성전자 → `005930`)
  - 미장: 대문자 티커 (예: 애플 → `AAPL`)
- 회사명만 받으면 `WebSearch`로 정확한 티커 먼저 확인

### 2. 데이터 수집 (병렬 실행)

**국장 종목 (한국)**

> ⚠️ **`finance.naver.com` 도메인 전체가 WebFetch에서 차단됨** — 아래 대체 출처 사용 필수.

- 시세/밸류에이션: `WebFetch` → `https://www.investing.com/equities/{영문-회사명-slug}` (PER/PBR/52주 고저/시총/현재가)
- 재무·컨센서스 종합: `WebFetch` → `https://comp.fnguide.com/SVO2/asp/SVD_Consensus.asp?gicode=A{종목코드}` (FnGuide)
- 증권사별 목표주가/리포트: `WebFetch` → `https://markets.hankyung.com/stock/{종목코드}/consensus` (한경)
- 리포트 검색 보강: `WebSearch` → `"{회사명}" 목표주가 컨센서스 {연도}` 또는 `"{회사명}" site:consensus.hankyung.com`
- 공시(필요시): `WebSearch` → `"{회사명}" site:dart.fss.or.kr`
- 최근 뉴스: `WebSearch` → `{회사명} 뉴스 {YYYY년 M월}` (반드시 현재 연·월 명시)

> 💡 Investing.com URL slug를 모르면 먼저 `WebSearch` → `"{회사명} site:investing.com"`으로 정확한 URL 확인.
> 💡 한경 컨센서스의 "평균 목표가"는 구버전 리포트가 섞여 낮게 나올 수 있음 → FnGuide·Investing 평균을 우선 신뢰하고, 한경은 **증권사별 개별 리포트 확인용**으로만 사용.

**미장 종목 (미국)**
- 시세/요약: `WebFetch` → `https://finance.yahoo.com/quote/{TICKER}`
- 재무: `WebFetch` → `https://finance.yahoo.com/quote/{TICKER}/financials`
- 애널리스트 컨센서스: `WebFetch` → `https://finance.yahoo.com/quote/{TICKER}/analysis`
- 리서치/등급: `WebSearch` → `{TICKER} analyst rating price target` / `{TICKER} site:seekingalpha.com`
- 공시(필요시): `WebSearch` → `{TICKER} 10-K site:sec.gov`
- 최근 뉴스: `WebSearch` → `{TICKER} news` (날짜 한정)

**차트/기술적 분석 (양 시장 공통)**
- 미장 기술적 요약: `WebFetch` → `https://www.investing.com/equities/{slug}-technical` (이동평균, RSI, MACD, 스토캐스틱, 매수/매도 시그널 요약)
- 미장 차트·핵심지표 한 페이지: `WebFetch` → `https://finviz.com/quote.ashx?t={TICKER}` (52주 차트·기술적 신호·뉴스 헤드라인 일체)
- 국장 기술적 분석: `WebFetch` → `https://www.investing.com/equities/{slug}-technical` (국장 종목도 Investing.com에 기술적 페이지 존재)
- 백업: `WebSearch` → `"{회사명}" 기술적 분석 RSI 이동평균` (애널리스트 차트 분석 글)

**동종업계 비교 (양 시장 공통)**
- 미장 피어 비교: `WebFetch` → `https://finviz.com/quote.ashx?t={TICKER}` 페이지 하단 peer list, 또는 `https://stockanalysis.com/stocks/{ticker}/competitors/`
- 미장 섹터 멀티플: `WebFetch` → `https://www.investing.com/equities/{slug}-ratios`
- 국장 피어 비교: `WebFetch` → `https://comp.fnguide.com/SVO2/asp/SVD_Corp.asp?gicode=A{종목코드}` (FnGuide "동일업종 비교")
- 피어 자동 식별: `WebSearch` → `"{회사명}" 경쟁사` / `{TICKER} competitors peers`
- 피어 식별 후, 각 피어에 대해 시세/PER/PBR/ROE/성장률을 **병렬 WebFetch**로 수집 → 비교 테이블 생성

> **Tip**: WebFetch는 항상 여러 페이지를 **병렬로** 호출해 시간 절약. 피어 3-5개의 데이터도 한 번에 병렬 fetch.

### 3. 리포트 구성

아래 7개 섹션을 **마크다운**으로 출력. 회사명·시장·티커를 헤더에 명시.

```markdown
# {회사명} ({티커}) — {시장} 분석 리포트
*분석 기준일: {YYYY-MM-DD}*

## 1. 회사 개요
- 업종 / 주요 사업
- 시가총액 / 발행주식수
- 사업 모델 한 줄 요약

## 2. 시세 & 재무 스냅샷
- 현재가 / 전일대비 / 52주 고저
- PER / PBR / 배당수익률
- 매출·영업이익·순이익 (최근 3년 또는 분기)
- 부채비율, ROE 등 핵심 지표

## 3. 애널리스트 컨센서스
- 컨센서스 의견 (매수/보유/매도 분포)
- 평균 목표주가 & 현재가 대비 상승여력 (%)
- 주요 증권사별 의견 (가능한 경우 3-5곳)
- 최근 의견 변경 (상향/하향 조정)

## 4. 차트 & 기술적 분석
- **추세**: 5/20/60/120일 이동평균 위치 관계 (정배열/역배열/꼬임)
- **모멘텀 지표**: RSI(14), MACD, 스토캐스틱 → 과매수/과매도/중립 판단
- **거래량**: 평균 대비 최근 거래량 흐름
- **종합 시그널**: Strong Buy / Buy / Neutral / Sell / Strong Sell (Investing.com 기술적 요약 활용)
- 가능하면 시각 차트 URL(finviz, TradingView 등) 링크로 첨부

### 지지·저항선 (필수 — 항상 다음 구조로 출력)

**현재가 기준으로 위/아래 각 3-5단계씩 정리한다.** 단일 가격이 아닌 **강도별·근거별** 다단계 구조.

**저항선 (위쪽 — 상승 시 막힐 가능성)**

| 강도 | 가격 | 거리 | 근거 |
|---|---|---|---|
| 🔴 강함 | 가격 | +X% | 52주 최고 / 강한 이전 고점 |
| 🟠 중간 | 가격 | +X% | MA50/MA200 / 직전 박스 상단 |
| 🟡 약함 | 가격 | +X% | 일중 고가 / R1 피봇 / 라운드 넘버 |

**지지선 (아래쪽 — 하락 시 받쳐줄 가능성)**

| 강도 | 가격 | 거리 | 근거 |
|---|---|---|---|
| 🟢 약함 (즉시) | 가격 | -X% | MA5/MA10 / 일중 저가 / S1 피봇 |
| 🟡 중간 | 가격 | -X% | MA20/MA50 / 라운드 넘버 / 직전 저항이었던 라인 |
| 🟠 강함 | 가격 | -X% | MA200 / 직전 박스 상단 / 강한 매수세 있었던 가격대 |
| 🔴 마지막 보루 | 가격 | -X% | 52주 최저 / 장기 박스권 중심 |

**시각화 (선택, 추세 직관 보조)**
간단한 아스키 차트로 현재가 기준 가격대를 위→아래 정렬해 한눈에 보이게:
```
저항 (가격) ─── 근거
저항 (가격) ─── 근거
🔵 현재가
지지 (가격) ─── 근거
지지 (가격) ─── 근거
```

**분기점 해석 한 줄 (필수)**
지지·저항 표 직후, **"단기 분기점은 X원 / 중기 분기점은 Y원"** 형식으로 핵심 임계가만 콕 집어 명시한다. 어디서 매수/매도세가 본격 전환될지 직관적으로 알 수 있도록.

## 5. 동종업계 비교
- 피어 3-5개 선정 근거 한 줄 (같은 섹터·유사 시총·직접 경쟁자 기준)
- 핵심 지표 테이블:

| 종목 | 시총 | PER | PBR | ROE | 매출성장률(YoY) | 영업이익률 | YTD 수익률 |
|---|---|---|---|---|---|---|---|
| **{본 종목}** | ... | ... | ... | ... | ... | ... | ... |
| 피어 1 | ... | ... | ... | ... | ... | ... | ... |
| 피어 2 | ... | ... | ... | ... | ... | ... | ... |

- **상대 평가 한 줄**: 본 종목이 피어 대비 저평가/적정/고평가인지, 어떤 지표가 차별화되는지

## 6. 최근 이슈 & 뉴스 (최근 1-3개월)
- 핵심 이벤트 3-5개 (날짜 포함)
- 각 이슈가 주가에 미친 영향 한 줄

## 7. 종합 의견
- 강점 (Bull case) 2-3개
- 약점/리스크 (Bear case) 2-3개
- 단기/중장기 관전 포인트
- **마지막 줄에 반드시 면책 문구 포함**
```

> 💡 **간략 모드**: 사용자가 "간단히", "요약만", "빠르게" 같은 표현을 쓰면 차트(4) + 동종업계(5) 섹션은 생략하고 5섹션 리포트(기존 1·2·3·6·7)로 단축한다.
> 💡 **단독 요청**: 사용자가 "차트만", "경쟁사만" 같이 특정 섹션만 요청하면 해당 섹션만 출력한다.

### 4. 면책 문구 (필수)
리포트 마지막에 반드시 다음을 포함:

> ⚠️ 본 분석은 공개 정보를 기반으로 한 참고 자료이며, **투자 권유나 매매 추천이 아닙니다.** 모든 투자 결정과 그 결과의 책임은 투자자 본인에게 있습니다.

### 5. 리포트 자동 저장 + Git 커밋 + Push

`~/personal/stock/reports/` 디렉토리가 존재하면, **리포트 출력 후 자동으로 저장 + git commit + push까지 진행한다**.

**파일명 규칙**:
- 기본: `YYYY-MM-DD_{종목코드 또는 티커}_{영문 회사명 소문자}.md`
  - 예: `2026-05-17_000660_skhynix.md`, `2026-05-17_NVDA_nvidia.md`
- **차트 단독 모드**: 끝에 `_chart` 접미사
  - 예: `2026-05-18_020150_lotte-energy-materials_chart.md`
- **동일 종목 같은 날 재분석**: 뒤에 `_v2`, `_v3` 추가
- **간략 모드/일부 섹션만**: 끝에 `_brief` 또는 해당 섹션 키워드 접미사

**자동 실행 흐름** (리포트 출력 직후):
1. 파일 저장: `~/personal/stock/reports/{파일명}.md`
2. Git commit: `cd ~/personal/stock && git add reports/ && git commit -m "Add {회사명} ({티커}) {분석유형} {날짜}"`
3. Git push: `git push`
4. 사용자에게 결과 짧게 알림: 파일 경로 + commit hash + GitHub URL

**자동 저장을 건너뛰는 경우** (다음 신호 감지 시):
- 사용자가 "저장하지마", "저장 X", "임시", "테스트", "비교용", "don't save", "save X" 등 명시적 거부
- 사용자가 메시지에서 "잠깐", "그냥 궁금해서" 같은 가벼운 톤 명시
- 분석이 풀 리포트가 아닌 단순 질문 응답에 그친 경우 (예: "현재가만 알려줘")

**판단이 모호한 경우**: 자동 저장 진행 후, 메시지 끝에 "저장 안 하실 거면 알려주세요"로 짧게 안내

## 출력 규칙

- **출처를 항상 명시**: 각 섹션 끝에 `(출처: 네이버금융, Yahoo Finance, ...)` 형식
- **수치는 최신 데이터**: 데이터가 며칠 이상 지난 경우 "OO일자 기준" 명시
- **추측 금지**: 컨센서스나 재무 데이터를 찾지 못하면 "공개 데이터 없음"으로 솔직히 표기
- **링크 포함**: 주요 출처 URL은 마크다운 링크로 첨부 → 사용자가 직접 검증 가능

## 주의사항

- **실시간 시세 아님**: WebFetch/WebSearch는 캐시된 페이지일 수 있음 → 정확한 매매가는 증권사 HTS 참고 권장
- **민감한 추천 회피**: "지금 사세요/파세요" 같은 직접 권유 표현 금지. "관전 포인트", "리스크 요인" 같은 분석적 표현 사용
- **언어**: 사용자가 한국어로 물으면 한국어로, 영어로 물으면 영어로 답변
- **데이터 충돌 시**: 출처별 수치가 다르면 둘 다 표기하고 출처 명시

## 알려진 도메인 차단/이슈

WebFetch가 차단되거나 동작이 불안정한 도메인 — 처음부터 대체 출처를 사용한다:

| 차단 도메인 | 용도 | 대체 출처 |
|---|---|---|
| `finance.naver.com` (전 페이지) | 국내 시세/재무/컨센서스/리서치 목록 | FnGuide(`comp.fnguide.com`), 한경(`markets.hankyung.com`), Investing.com |
| `*.naver.com` 계열 일반 | 뉴스/검색 | WebSearch로 대체 |

> 새로운 차단 도메인을 발견하면 이 표에 추가하고 대체재를 기록할 것.

## 확장 옵션 (요청 시)

사용자가 추가로 요청할 수 있는 심화 분석:
- **공시 요약**: 최근 분기보고서/사업보고서 핵심 변경사항 (DART/SEC EDGAR)
- **배당 이력**: 최근 5년 배당 추이 및 배당성장률
- **수급 분석**: 외국인·기관 매매 동향 (국장), 13F·기관 보유율 (미장)
- **숏 인터레스트**: 미장 한정 — 공매도 비율, days to cover
- **옵션 체인**: 미장 한정 — 주요 행사가 IV, Max Pain
